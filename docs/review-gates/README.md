# 代码交付门禁索引

本目录维护代码交付前的检视规范。
根 `AGENTS.md` 只保留门禁入口；具体范围、工具和失败处理写在本目录。

## 判定表

| 场景 | 需要检视 | 规范 |
| --- | --- | --- |
| C/C++ 代码行为、BUILD、接口/IPC 或跨模块逻辑 | 稳定性 | `stability.md` |
| IPC、不可信输入、权限或安全边界 | 稳定性；安全规范待补 | `stability.md` |
| 纯文档、注释、知识路由、格式调整 | 通常不适用 | 最终回复说明不适用原因 |

## 使用规则

- 默认按本次任务影响面选择检视项，不默认全仓扫描。
- 若多个门禁同时适用，按相关规范分别说明结果。
- 门禁结果在最终回复中标注为：已执行、失败、不可用或不适用。
- 若 Agent 负责 commit 或 push，相关门禁失败或不可用时不继续 push，等待人工确认。
- 新增其他检视规范时，在本目录新增对应 `.md` 并更新判定表。
