# 新增多语言公开 API

适用于新增或变更 NAPI、NDK、CJ、ANI、Taihe 或 Native 公开 API 行为。

执行要点：

- 先确认 `interfaces/` 声明、错误码映射、默认值、权限和 XTS 预期。
- 不要只改某一层语言绑定改变公开行为；同步检查 JS、NDK、CJ、ANI、Taihe 和 Native 入口。
- 检查实现层、测试层和文档/声明是否保持一致。
- 涉及异常输入、空对象、边界值和权限失败时，复用附近错误码和日志约定。

门禁：

- 改 ABI、枚举、结构体字段、错误码、权限或默认行为前，必须人工确认兼容策略。
- 涉及跨语言公开行为时，最终回复说明已检查哪些语言入口，哪些入口不适用。

验证建议：

- 跑对应语言单测。
- 涉及公开行为时验证或说明 XTS 目标。
- 无法跑 XTS 或设备验证时，在最终回复列出缺口和需要人工确认的问题。
